PREMIER JET — [ Nom a Determiner ] — Roguelite Auto-Battler 2D

Jeu mobile + PC, 2D, inspire de Forge Master / idle RPG roguelite. Solo dev.

1. Concept du Jeu

Pitch

Roguelite auto-battler 2D. Le joueur gere une equipe d'aventuriers qui combat des vagues d'ennemis par paliers. Chaque dimanche minuit la run reset : on perd aventuriers, loot et paliers, mais on garde les ameliorations permanentes (batiments, arbre de competences, benedictions). Le jeu farm meme offline.

Boucle de jeu

Recruter (village) → equiper → combat auto + skills manuels → loot → ameliorer batiments → reset dimanche → recommencer plus fort.

Farm offline

Quand le joueur quitte, le serveur simule les combats au retour (basé sur le temps ecoule depuis la derniere connexion). Le loot et la progression sont calcules côte serveur.

Mecanique cle : le reset progressif

Les ameliorations de batiments faites pendant une run ne sont qu'a 50% d'effet. Au reset dimanche, elles passent a 100%.

Exemple : ameliorer le batiment de recrutement passe de 100% rang F a 95%F / 5%E. Mais pendant la run c'est ~97%F / 3%E (50%). Apres reset → 90%F / 10%E complet.

Version Desktop — Mode Overlay

La version PC sera idealement en mode borderless overlay / always-on-top : fenetre sans bordure, toujours au premier plan, meme par-dessus un jeu en plein ecran. Cela permet de farm pendant qu'on joue a autre chose. Si techniquement impossible (selon l'OS/drivers), fallback sur un simple mode borderless window classique (sans bords, mais pas overlay).

A investiguer : Unity supporte le borderless natif. Pour l'overlay always-on-top par-dessus les jeux fullscreen, il faudra potentiellement un plugin natif Windows (SetWindowPos / TOPMOST) ou un wrapper. A tester en V1.

2b. Strategie Backend — Phase Client-Only (V1-V4)

V1 a V4 : pas de backend. Toutes les donnees sont mockees ou en dur cote client Unity. Le vrai backend (ASP.NET Core + PostgreSQL) sera implemente en V5.

Cette strategie permet de developper et tester 100% du gameplay sans infrastructure serveur. Les interfaces (Services) seront concues pour etre facilement remplacees par des appels API en V5 (pattern Strategy ou injection de dependances).

5. Batiments du Village — Detail

Chaque batiment est upgradeable. Effet a 50% pendant la run, 100% apres reset. Accelerable avec gems.

AspectDetail

5.3 Entrepot (Batiment Loot / Inventaire)

Batiment central pour la gestion du loot. Plusieurs niveaux d'upgrade qui debloquent des fonctionnalites.

FeatureDetail
Taille inventaireNombre de slots de loot. Upgradeable. Important pour le farm offline.
Auto-vente basiqueSi inventaire plein et nouveau loot : compare l'item level, vend automatiquement le plus faible.
Auto-vente intelligenteUpgrade : vendre auto les items en dessous d'une rarete minimum (ex: vend tout en dessous de 5 etoiles).
Filtre par statsUpgrade : vendre auto les items qui n'ont pas certaines stats souhaitees.
Filtre par item levelUpgrade : seuil minimum d'item level, en dessous = vente auto.

5.4 Forge (Batiment Amelioration Items)

Batiment dedie a l'amelioration des stats des items eux-memes.

FeatureDetail
Augmenter stats maxAugmente le plafond des stats que les items peuvent avoir.
Diminuer stats minReduit le plancher (meilleur worst case sur les rolls).
Rarete = multiplicateurLa rarete de l'item multiplie les stats OU augmente le max / diminue le min.
Stats supplementairesUpgrade : un item peut proposer 1 stat de plus en echange d'un max reduit. Ex: au lieu de 1 stat entre 20-100%, on a 2 stats entre 10-60%.
Debloquer stats cacheesCertaines stats ne sont pas visibles par defaut sur les items. La forge permet de les reveler.

Tout est upgradeable par niveau. Chaque palier de forge debloque ou ameliore une de ces features.

5.5 Temple des Benedictions (Batiment Traits)

Systeme pour influencer les traits des futurs aventuriers recrutes.

AspectDetail
TraitsChaque aventurier a des traits aleatoires (positifs et negatifs). Ex: "+30% degats lance", "-10% PV".
Rarete des traitsCommun, Rare, Epique, Legendaire. Probas augmentables.
BenedictionRassembler plusieurs aventuriers avec le MEME trait pour activer une benediction sur ce trait.
CoutPar defaut : 5 aventuriers avec le meme trait requis. Upgradeable pour diminuer ce nombre.
EffetLe trait beni apparait plus souvent chez les futurs recrutes — pendant la run actuelle ET les suivantes.
LimiteNombre max de benedictions actives simultanées. Augmentable par upgrade.
Pas de sacrificeLes aventuriers ne sont pas detruits. On les garde. Il faut juste en avoir assez avec le bon trait.

6. Gameplay — Synthese

6.1 Aventuriers

Classes fixes (Guerrier, Tank, Mage, Healer, Archer) avec stats aleatoires controlees (min/max par classe). Rangs F→S. Chaque aventurier a des traits positifs et negatifs aleatoires.

6.2 Combat

Semi-auto : avancent et attaquent seuls, skills activables manuellement. Ennemis par paliers. Mort = retour palier precedent. Farm offline simule par le serveur.

6.3 Loot

Drop par monstre, influence par palier + luck du compte. Gere par l'Entrepot (auto-vente configurable). Inventaire visible dans l'ecran combat.

6.4 Traits & Benedictions

Traits sur aventuriers uniquement. Rarete variable. Benissables via le Temple pour augmenter les probas (run actuelle + futures). Nombre de benedictions limite et upgradeable.

6.5 Reset hebdo

Dimanche minuit : paliers → 1, aventuriers perdus, loot perdu. Batiments + arbre + benedictions conserves. Upgrades en cours passent de 50% → 100%.

7. Roadmap

V1 Socle Minimal

100% client, donnees mockees. 1 guerrier (ATK, PV, regen PV, vitesse attaque), avance gauche→droite, 1 ennemi par palier, palier suivant si tue / retour si mort. HUD bas avec stats + equipement. Loot basique au kill.

V2 Equipe & Classes

Centre de Recrutement + Caserne. 5 classes, rangs F→S, stats aleatoires controlees, bonus/malus. Equipe upgradeable. Skills manuels. Plusieurs ennemis par palier.

V3 Economie & Batiments

Monnaie in-game + gems. Batiments upgradeable (Caserne, Forge, Entrepot, Recrutement). Mecanique 50%/100%. Reset hebdo dimanche. Entrepot avec auto-vente.

V4 Traits, Benedictions & IA

Systeme de traits. Temple des Benedictions. Centre Entrainement (niveaux IA). Forge avancee (stats cachees, stats bonus). Arbre de competences permanent.

V5 Monetisation & Online

Backend reel (ASP.NET Core + PostgreSQL). Remplacement des mocks par les vrais services API. Auth complete + sauvegarde serveur + anti-triche. Farm offline (simulation serveur). Leaderboard hebdo. Pubs rewarded, pass anti-pub, cosmetiques+stats, shop gems.

V+ Futures

Guildes, events, boss hebdo, PvP indirect, nouveaux batiments, nouvelles classes...
